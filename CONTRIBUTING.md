# Как внести вклад в FilmLightMeter

Спасибо, что хотите помочь сделать приложение лучше. FilmLightMeter — открытый проект под MIT, и ему нужны сообщения о багах, идеи, калибровочные данные для новых телефонов и пресеты плёнок/камер/объективов.

Этот файл объясняет, как сообщить о проблеме, предложить улучшение и оформить pull request так, чтобы его было легко принять.

## 🐛 Сообщить об ошибке

Самый ценный вклад — подробный баг-репорт. Откройте [новый issue](https://github.com/Dnlln/FilmLightMeter/issues/new/choose) и выберите шаблон **«🐛 Баг-репорт»**.

**Хороший баг-репорт содержит:**

- Версию приложения (`Настройки → О программе` или по тегу релиза)
- Модель телефона и версию Android
- Плёнку, камеру и объектив, которые выбраны в приложении
- Условия съёмки: EV₁₀₀ на экране, сцена (солнце/пасмурно/помещение)
- Что вы ожидали и что произошло на самом деле
- Скриншот — кнопка **«Снять»** в приложении сохраняет его в галерею одним тапом

Перед тем как открыть issue, проверьте [уже существующие](https://github.com/Dnlln/FilmLightMeter/issues?q=is%3Aissue) — возможно, о проблеме уже сообщили.

## 💡 Предложить улучшение

Идеи новых функций, режимов замера, интерфейсных удобств — через шаблон **«💡 Запрос на фичу»**. Опишите:

- Какую задачу это решает (конкретный сценарий съёмки)
- Почему текущий интерфейс не справляется
- Как вы видите решение (не обязательно в деталях)

## 🎞️ Предложить пресет плёнки/камеры/объектива

Для плёнок, камер и объективов есть шаблон **«🎞️ Новый пресет»**. Укажите:

- **Плёнка:** название, ISO, тип (негатив/слайд/ч-б), экспонент закона взаимности (если известен — из техпаспорта)
- **Камера:** модель, список выдержек (`1/30, 1/60, …`), наличие режима B
- **Объектив:** название, фокусное, светосила, полный список значений диафрагмы (как нарисованы на кольце)

Ссылайтесь на источник (техпаспорт, мануал, Photrio, APUG), если это не очевидные данные.

## 📐 Калибровочные данные для телефона

Если ваш телефон систематически даёт EV на X стопов в сторону от реального (проверено по цифровой зеркалке или Sekonic) — откройте issue с пометкой `calibration`. Укажите модель, версию Android, сдвиг в стопах и условия, где вы это проверяли. В будущем такие данные будут включены в таблицу автокалибровки.

## 🔧 Pull Requests

PR приветствуются. Рекомендованный порядок:

1. **Обсудите крупное изменение в issue** перед кодированием — сэкономит время обоим
2. **Форкните** репозиторий и сделайте ветку от `main`: `git checkout -b feature/incident-metering`
3. **Держите PR компактным** — одна фича или один баг на PR. Несвязанные правки лучше разделить
4. **Проверьте локальную сборку:**
   ```bash
   ./gradlew :app:assembleDebug
   ```
   APK должен собраться без предупреждений
5. **Следуйте стилю кода:**
   - Kotlin official style (Android Studio форматирование по умолчанию)
   - UI на Jetpack Compose, логика — в `ViewModel`, математика экспозиции — в `ExposureMath`
   - Пресеты (плёнки, камеры, объективы) — отдельными объектами в `data/*Presets.kt`
   - Комментарии и имена — на русском или английском, внутри одного файла выдерживать один язык
6. **Заполните шаблон PR** — объясните *что* и *зачем*. Если поменяли UI, приложите скриншот/видео
7. **Один коммит = одно осмысленное изменение.** Squash перед merge приветствуется

### Тесты

Юнит-тестов пока нет (проект маленький). Если добавляете математику — добавьте юнит-тест под `app/src/test/`. Для проверки формул экспозиции тест-кейсы ищите в [ISO 2720](https://www.iso.org/standard/7690.html) и [APEX specification](https://en.wikipedia.org/wiki/APEX_system).

### Конвенция коммитов

Свободная форма, но префиксы помогают читать историю:

- `fix:` — исправление бага
- `feat:` — новая функция
- `docs:` — только документация
- `refactor:` — рефакторинг без смены поведения
- `data:` — добавление/изменение пресетов

## 🗺️ Куда смотреть в коде

```
app/src/main/java/com/filmlightmeter/app/
├── exposure/ExposureMath.kt  ← формулы APEX, snapShutterToCamera, findBestPairs
├── data/
│   ├── FilmPresets.kt        ← 31 плёнка
│   ├── CameraPresets.kt      ← 9 камер
│   └── LensPresets.kt        ← 9 объективов
├── camera/                    ← Camera2, YUV-анализ
├── ui/
│   ├── MeterViewModel.kt     ← состояние + сглаживание EV
│   └── screens/MeterScreen.kt ← главный экран, диалог справки
└── util/ScreenCapture.kt     ← скриншот в галерею
```
